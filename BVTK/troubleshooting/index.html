Troubleshooting
Good morrow my dear friend, I'm sorry to say that if you got here, then something went wrong during your VTK installation. In this page you'll find a few terminal commands that could be useful to fix your problem. Meanwhile, if you want you can open an issue on our GitHub page so that we can try to solve your error once and for all and make the life easier for other users.
Solution 1
Sometimes to fix the problem is enough closing and reopening Blender, then repeat the installation. Try doing this before going further, it could save you some time.
Solution 2
The first solution is to install VTK using pip on your Blender's Python. These are the same commands that the BVTK add-on tries to perform automatically, but running it in the terminal gives you a greater degree of freedom and permits you to see what is the error that is actually impeding the installation. So just open the terminal and run the commands, granting admin privileges if needed, then wait and check if something goes wrong.
-m ensurepip
-m pip install vtk
If pip doesn't throw any error you can open Blender again and hopefully, you should be able to enable the add-on without troubles. Otherwise, if the VTK installation fails, since the error could be due to thousands of things, you'll have to fix the problem by yourself. Most of the times the best strategy is to search the error online and look for a solution. If you can't get out of it, you may skip to the third solution.
Solution 3
The last solution you can try is to install on your machine the same Python version running inside Blender, then install VTK on this new Python. The disadvantage is that you will have to open Blender from the terminal each time you'll want to use BVTK (in fact opening Blender from the graphical interface wouldn't import VTK)
python3.5 -m ensurepip
python3.5 -m pip install vtk
If you manage to install VTK in this way you should be able to use BVTK, but only opening Blender from the terminal.
